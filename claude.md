# CLAUDE.md - Technical Documentation

This section provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Project Overview

**FixIt** is a gamified life transformation app built with Next.js that implements Dan Koe's behavior change methodology. The app guides users through a structured onboarding process (22 questions), then provides daily, weekly, and monthly routines to help them achieve personal transformation through a game-like experience with XP, levels, quests, and boss fights.

## Tech Stack

- **Framework**: Next.js 16 (App Router) with TypeScript
- **UI**: React 19 + Tailwind CSS v4 + Framer Motion
- **Backend**: Supabase (PostgreSQL, Auth, Real-time)
- **Deployment**: Vercel

The app uses Supabase with anonymous authentication for demo purposes. To set up:

   ```
   NEXT_PUBLIC_SUPABASE_ANON_KEY=your-anon-key
   ```
3. Run the schema in `supabase/schema.sql` via Supabase SQL Editor

## Architecture

### Core User Flow

The app follows a strict progression model enforced by `app/page.tsx`:

1. **Anonymous Auth** ‚Üí Automatically creates anonymous session on first visit
2. **Onboarding** ‚Üí 22-question psychological excavation if no character sheet exists
3. **Daily Check** ‚Üí Required daily direction check if not completed today
4. **Dashboard** ‚Üí Main game HUD with XP, level, and navigation

### Data Model (6-Component Framework)

The core of the app is the "Character Sheet" derived from onboarding questions:

- **Anti-Vision** (Q17): Stakes - what user refuses to become
- **Vision** (Q18): Win condition - what user is building toward
- **Mission** (Q19): 1-year goal
- **Boss** (Q20): 1-month project
- **Quests** (Q21): 2-3 daily levers/actions
- **Rules** (Q22): Constraints user won't sacrifice

### Key Components

**Onboarding Flow** (`components/onboarding/OnboardingFlow.tsx`)
- Multi-step form with 22 questions from Dan Koe's protocol
- Sections: Pain Awareness (Q1-4), Anti-Vision (Q5-11), Vision (Q12-14), Synthesis (Q15-22)
- Saves responses to `onboarding_responses` and generates `character_sheet` + `daily_levers`

**Game HUD** (`components/dashboard/GameHUD.tsx`)
- Shows current level, XP, progress bar, streak
- Navigation to routines and character sheet
- Menu modal for settings/reset

**Daily Direction Check** (`components/routines/DailyDirectionCheck.tsx`)
- 5-minute routine: swipe vision/hate direction + comment
- Check off daily levers (quests)
- Awards XP, updates streak
- Creates/updates `daily_logs` entry

**Weekly Reflection** (`components/routines/WeeklyReflection.tsx`)
- 8-minute review of past 7 days
- Pattern analysis, anti-vision reminder, lever adjustment, project progress
- Awards +200 XP

**Monthly Boss Fight** (`components/routines/MonthlyBossFight.tsx`)
- 10-minute project review
- Mark boss defeated/failed, set new monthly project
- Awards +1000 XP for completion, +250 XP for attempt

**Character Sheet** (`components/dashboard/CharacterSheet.tsx`)
- Displays all 6 components in game-styled UI
- Edit mode to update any component
- Visual representation of user's framework

### Database Tables

- `users` - User profiles with XP, level, streaks
- `character_sheet` - 6-component framework (1:1 with users)
- `daily_levers` - User's active daily quests
- `daily_logs` - Daily check-in records
- `onboarding_responses` - Raw answers to 22 questions
- `weekly_reflections` - Weekly review data
- `boss_fights` - Monthly project tracking
- `achievements` - Unlocked achievements

### XP System

**Earning XP:**
- Daily lever complete: +25-100 XP (user-defined per lever)
- Daily direction check (vision): +50 XP
- Weekly reflection: +200 XP
- Monthly boss defeated: +1000 XP / failed: +250 XP
- Streak bonuses: 7 days (+500), 30 days (+2000), 90 days (+5000)

**Level Progression:**
1. Conformist (0-500 XP)
2. Self-Aware (500-1500 XP)
3. Architect (1500-3500 XP)
4. Builder (3500-7500 XP)
5. Strategist (7500-15000 XP)
6. Visionary (15000-30000 XP)
7. Master (30000-60000 XP)
8. Legend (60000+ XP)

Defined in `types/index.ts` as `LEVEL_TITLES` and `XP_THRESHOLDS`.

### State Management

The app uses local React state with Supabase for persistence. The main routing logic in `app/page.tsx` determines which view to show based on:

1. User authentication status
2. Character sheet existence
3. Daily check completion

Navigation between views is controlled via `setCurrentView('view-name')`.

## Important Patterns

### Demo Mode with Anonymous Auth

The app uses Supabase anonymous authentication for demo purposes. On page load, `checkAuthAndProgress()` either retrieves an existing session or creates a new anonymous one. This allows users to try the app without sign-up.

### Question-to-Component Mapping

The 22 onboarding questions map to specific parts of the character sheet. Key mappings in `lib/questions.ts`:

- Q17 ‚Üí Anti-Vision
- Q18 ‚Üí Vision
- Q19 ‚Üí Year Goal
- Q20 ‚Üí Month Project
- Q21 ‚Üí Daily Levers (split by newlines)
- Q22 ‚Üí Constraints

### Daily Check Logic

The daily check is required once per day. The app checks if a `daily_logs` entry exists for today's date (`YYYY-MM-DD`). If not, user is routed to `DailyDirectionCheck` before accessing dashboard.

### XP Calculations

XP is calculated dynamically based on actions. When daily levers are checked off, their individual `xp_value` is summed. Level is derived from `total_xp` by comparing against `XP_THRESHOLDS`.

## Common Workflows

### Adding a New Routine

1. Create component in `components/routines/`
2. Add view type to `View` union in `app/page.tsx`
3. Add navigation logic to `GameHUD.tsx` menu
4. Create database table/columns if needed
5. Update types in `types/index.ts`

### Modifying Questions

Questions are defined in `lib/questions.ts` as `ONBOARDING_QUESTIONS` array. Each has:
- `number`: Question number (1-22)
- `text`: Exact wording from Dan Koe
- `section`: pain | anti-vision | vision | synthesis
- `placeholder`: Optional input hint

After modifying, ensure `OnboardingFlow.tsx` extraction logic still maps correctly to character sheet.

### Adjusting XP/Levels

1. Edit `XP_THRESHOLDS` in `types/index.ts`
2. Add/remove levels in `LEVEL_TITLES`
3. Update README.md game mechanics section
4. Consider migration for existing users if thresholds change

### Resetting User Data

The app includes a reset function in `MenuModal.tsx` that:
1. Deletes all user data from Supabase tables
2. Signs out anonymous user
3. Refreshes page to restart onboarding

## Testing

## Development Philosophy

This app is built around Dan Koe's "Turn Your Life Into A Video Game" methodology. When making changes:

- Preserve the exact wording of the 22 questions
- Maintain the game metaphor (quests, bosses, XP, loot)
- Keep routines time-bound (5 min daily, 8 min weekly, 10 min monthly)
- Respect the 6-component framework as the core data model
- Prioritize user momentum over feature bloat
